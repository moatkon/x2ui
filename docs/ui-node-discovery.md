# X2Post 节点发现入口升级

> 日期：2026-07-13  
> 范围：首页节点筛选、节点目录、节点详情互通与 List 分隔线  
> 依据：`docs/ux-research.md`、`轻量级社区_UI设计描述文档.md`、daisyUI 5.6.x

## 目标

让匿名访客在进入首页后的 10 秒内理解：X2Post 按“节点”组织长期讨论，能够先缩小兴趣范围，再进入帖子；同时明确节点与标签的差异。

本次不增加节点管理、推荐、热门或运营精选能力。所有节点均来自 X2API 的公开只读节点接口。

## 组件决策

| 候选 | 结论 | 原因 |
|---|---|---|
| Menu | 首页全断点节点导航采用 | “全部 + 5 节点”直接可见，适合在不同帖子视图间导航；320px 自然换行，当前项使用 `menu-active` 与 `aria-current` |
| Select | 不采用 | 虽然紧凑，但会把核心节点入口藏进下拉，低触达，不符合节点可发现性目标 |
| Filter | 不采用 | daisyUI Filter 选中后会隐藏其他选项，仅保留选中项与重置；这与“提高节点可发现性”的目标冲突 |
| Tabs | 不采用 | Tabs 用于同一对象的内容分区，节点是可独立访问的讨论空间，不应伪装成页面 Tab |
| Dropdown | 不作为主入口 | 会隐藏节点选项，适合次级菜单而非首次发现 |
| List | 节点目录与帖子结果继续采用 | 连续信息行比 Card 更适合比较名称、描述、帖子量和关注量 |

## 已实现的信息路径

```text
首页
├─ 按节点浏览（桌面/移动均为可见 Menu，小屏自然换行）
│  ├─ 全部帖子
│  └─ 节点帖子筛选 → 帖子详情
├─ 了解全部节点 → 节点目录
└─ 右栏节点/标签简短解释 → 全部节点

节点目录
├─ 解释节点与标签区别
├─ 节点详情
└─ 只看此节点帖子 → 首页筛选态

节点详情
├─ 当前节点规则与关注
├─ 当前节点帖子
└─ 全部节点
```

## 交互规则

- 首页使用 `#/feed?node={slug}` 表达可复制、可后退的筛选状态。
- 当前节点同时通过标题、筛选控件选中态、结果数量和清除筛选入口表达，不只依赖颜色。
- 无帖子节点显示原位 Empty 状态，并提供“查看节点”和“清除筛选”两个恢复动作。
- 节点目录每行同时提供“进入节点详情”和“只看此节点帖子”，不增加任何写管理入口。
- 首页右栏只解释节点与标签的区别并提供“全部节点”，不重复展示前三个节点。
- 所有断点都使用普通链接，键盘与浏览器历史行为天然保留；小屏快捷项自然换行，不能造成横向滚动或整页溢出。

## List 分隔线修复

原实现依赖父级 `divide-y-2`。为避免 daisyUI `list-row` 布局或浏览器生成样式造成分隔线不连续，改为由父 List 通过 Tailwind 子选择器直接给所有非末行的 `list-row` 添加 `border-bottom`：

- 外框仍只存在于 List 父级。
- 分隔线附着于完整行盒，横向贯通父容器内缘。
- 最后一行无多余底线。
- 回复或内部内容缩进不会影响行分隔线。
- 行本身保持无圆角、无阴影、无左右 margin 和行间 gap。

## 验收要点

- [ ] `#/feed` 显示全部 4 篇演示帖子，节点控件显示“全部”。
- [ ] `#/feed?node=product` 只显示产品设计帖子，标题和结果摘要同步更新。
- [ ] `#/feed?node=life` 显示可恢复空状态，不出现推荐或假内容。
- [ ] 桌面节点 Menu 的当前项具有文字可见状态及 `aria-current="page"`。
- [ ] 移动端直接显示“全部 + 5 节点”快捷项，可切换任一节点；320px 自然换行且无整页横向滚动。
- [ ] 首页右栏解释节点/标签区别，节点目录和节点详情均可进入全部节点。
- [ ] 节点目录明确说明节点是长期空间、标签是跨节点话题。
- [ ] 不存在节点新增、编辑、排序、删除入口。
- [ ] 所有连续 List 的分隔线贯通、末行无额外底线，浅色和深色主题均清晰。
- [ ] Node 语法检查通过，其他页面路由与交互保持不变。
