# X2Post 静态原型 UI 设计与全页面验收清单

> 角色产物：UI Designer  
> 日期：2026-07-13  
> 依据：`轻量级社区产品方案.md`、`轻量级社区_UI设计描述文档.md`、daisyUI 5.6.x 技能规范  
> 用途：约束本轮静态 HTML 原型的页面覆盖、交互跳转、响应式和验收；不替代产品/UI 事实源。

## 1. 不可变设计边界

- 所有可见品牌统一为 **X2Post**，生产 canonical origin 为 `https://x2post.com`。
- 本轮覆盖 P0 + P1 全闭环；不得出现推荐/热门、活动、徽章扩展、推广、私信、会员、钱包、订单、支付或订阅入口。
- 节点只读：没有新增、编辑、删除、排序或后台节点管理页面。
- 已发布帖子和评论不可编辑、删除、撤回、替换；任何身份都不出现这些操作。
- 治理只能隐藏/恢复可见性、锁定/解锁、限制/恢复用户和处理申诉，不得改写或删除原文。
- 公共页面匿名直达；受限操作先保存来源、滚动位置和输入，登录后恢复任务。
- 原型中的固定演示数据仅用于设计参考；生产页面必须读取 X2API，未接能力显示明确不可用或真实空态。

## 2. 信息架构

```text
X2Post
├─ 发现（公开）
│  ├─ 最新 Feed
│  ├─ 节点目录 → 节点详情/节点帖子
│  ├─ 搜索 → 帖子/用户/节点/标签
│  ├─ 标签详情
│  ├─ 帖子详情 → 评论/回复
│  └─ 用户公开页 → 帖子/评论/关注者/正在关注
├─ 账户（公开入口，私有内容）
│  ├─ 登录/注册
│  ├─ 邮箱验证/重发
│  ├─ 忘记密码/重置密码
│  ├─ 本人主页
│  └─ 设置 → 资料/隐私/通知/安全/设备会话
├─ 参与（登录）
│  ├─ 关注 Feed
│  ├─ 收藏
│  ├─ 通知
│  ├─ 草稿列表 → 草稿编辑/预览/发布
│  ├─ 举报进度
│  ├─ 屏蔽列表
│  └─ 申诉 → 新建/详情
└─ 治理（权限用户）

   ├─ 案件详情/治理动作
   ├─ 申诉处理
   └─ 审计日志
```

## 3. 全局导航

### 3.1 桌面 Navbar

- 左：X2Post Logo，点击回首页。
- 中：`首页`、`节点`；登录后增加 `关注`；搜索为显式入口。
- 右：匿名态显示 `登录`、`注册`；登录态显示唯一主操作 `发布`、通知未读入口、头像菜单。
- 头像菜单：`我的主页`、`收藏`、`草稿`、`举报进度`、`屏蔽列表`、`申诉`、`设置`、`退出登录`；具备治理权限时增加分组 `治理工作台`。
- 当前项通过文字、`aria-current="page"` 和非颜色视觉标记共同表达；普通导航不使用主色。

### 3.2 移动 Drawer

- `<768px` 时 Navbar 只保留菜单、Logo、搜索、通知或登录入口。
- Drawer 复用桌面完整导航；打开后焦点进入，Escape/遮罩关闭，关闭后焦点回到触发按钮。
- 页面核心内容位于 Drawer 的内容区；移动单栏中核心内容始终早于右栏上下文。

### 3.3 权限可见性

| 身份 | 可见全局入口 | 不可见入口 |
|---|---|---|
| 匿名 | 首页、节点、搜索、登录、注册 | 关注、发布、通知、私有菜单、治理 |
| 登录用户 | 匿名入口对应内容 + 关注、发布、通知、私有菜单 | 治理（无权限时） |
| 治理人员 | 登录用户全部入口 + 治理工作台 | 节点管理、内容编辑/删除 |

## 5. 页面内功能与弹窗/浮层清单

弹窗优先使用原生 `dialog` 封装的项目 `Dialog`；轻量菜单使用点击触发的 `DropdownMenu`；不得用 hover-only 交互。

| ID | 类型 | 触发位置 | 内容/结果 | 取消或失败行为 |
|---|---|---|---|---|
| O01 | Drawer | 移动 Navbar 菜单 | 完整导航 | 关闭后焦点恢复 |
| O02 | Dropdown | 登录用户头像 | 私有导航和退出 | 点外/Escape 关闭 |
| O03 | Dropdown | 帖子/评论反应 | 契约定义的反应类型与当前状态 | 失败恢复原计数/状态 |
| O04 | 建议浮层 | 编辑器输入 `@` | 允许被提及的用户建议 | 无结果/限流原位说明 |
| O05 | Dialog | 发布按钮 | 最终预览、节点/标题摘要、“发布后不能编辑或删除”确认 | 取消回编辑器且输入不丢失 |
| O06 | Dialog | 放弃草稿 | 草稿标题、不可恢复后果 | 取消回编辑器；失败保留草稿 |
| O07 | Dialog | 离开脏表单 | 保存并离开/放弃/继续编辑 | 默认焦点“继续编辑” |
| O08 | Dialog | 草稿版本冲突 | 本地/远端更新时间，重新加载或另存处理 | 禁止静默覆盖 |
| O09 | Dialog | 举报帖子/评论/用户 | 目标安全摘要、原因、说明、幂等提交 | 失败保留选择与说明 |
| O10 | Dialog | 屏蔽用户 | 对 Feed、搜索、提及、通知的影响 | 取消无变化 |
| O11 | Dialog | 解除屏蔽 | 关系恢复说明 | 失败保持已屏蔽 |
| O12 | Dialog | 撤销设备会话 | 设备、位置/时间、风险说明 | 当前会话采用更强确认 |
| O13 | Dialog | 全部标为已读 | 未读数量与后果 | 失败保留未读状态 |
| O14 | Dialog | 隐藏/恢复内容 | 只改可见性、必填理由、关联案件 | 失败不改变状态 |
| O15 | Dialog | 锁定/解锁帖子 | 评论影响、理由 | 失败不改变状态 |
| O16 | Dialog | 用户处置 | 警告/限流/禁言/封禁/恢复、期限、理由 | 动态校验，失败保留输入 |
| O17 | Dialog | 申诉决定 | 维持/撤销处置、理由 | 失败保持待处理 |
| O18 | Dialog | 会话失效 | 登录提示与可恢复任务摘要 | 去登录保存来源和输入 |
| O19 | Alert | 附件失败 | 类型/大小/扫描/配额原因、移除/重试 | 其他输入不丢失 |
| O20 | Toast | 非关键成功播报 | 已保存草稿、已复制链接等短反馈 | 不能替代错误恢复入口 |

评论/回复提交不强制二次弹窗，但提交按钮上方必须常驻“提交后不能编辑或删除”说明；提交中禁用重复提交，失败后保留正文。

## 7. 页面状态覆盖

### 7.1 一级模块通用状态

所有 Feed/List/Panel/Table/Form 必须在原模块边界内支持以下状态：

| 状态 | 静态原型表现 | 关键验收 |
|---|---|---|
| Loading | 原结构 Skeleton + 可读“正在加载” | 父区域 `aria-busy`，不引发布局大跳动 |
| Success | 正常数据/成功结果 | 写操作通过 live region 礼貌播报 |
| Empty | 单个全宽行 + 最多一个真实主操作 | 无嵌套卡片、无 P2 推荐填空 |
| Partial | 已加载内容 + 贯通 Alert + 局部重试 | 右栏失败不阻塞左栏 |
| Error | 原位 Alert、requestId、重试 | 不用 Toast 取代恢复入口 |
| Unauthorized | 任务摘要 + 去登录 | 保存来源、位置和输入 |
| Forbidden | 权限说明 + 可行去向 | 隐私契约时改为安全 404 |
| Not Found | 安全说明，不泄露隐藏内容 | 返回列表/首页 |
| Rate Limited | 原位说明 + 可重试倒计时 | 禁用重复动作直至允许 |
| Stale Target | 通知/举报目标不可访问 | 保留上下文但不泄露原文 |

### 7.2 写操作附加状态

- `Dirty`：未保存标志，离开触发 O07。
- `Validating`：字段旁加载文字，不锁死无关输入。
- `Submitting`：按钮 Loading、`aria-disabled`，幂等防重。
- `Conflict`：触发 O08，禁止静默覆盖。
- `Failed`：字段/模块内错误，保留全部安全输入。
- `Succeeded`：更新当前页面；只有页面目标变化时才跳转。

静态实现建议用 `?state=loading|empty|partial|error|unauthorized|forbidden|not-found|rate-limited|conflict` 显示每页状态，或提供仅供设计验收的 `prototype-index.html` 页面索引；状态切换器不得进入产品导航。

## 8. 组件决策（daisyUI 适配层）

已比较 Navbar/Menu/Drawer/Dropdown、List/Table/Card、Fieldset/Input/Textarea/Select/Validator、Tabs/Join、Modal/Alert/Toast/Loading/Skeleton 等候选组件：

| 场景 | 选择 | 选择理由/约束 |
|---|---|---|
| 全局导航 | Navbar + Menu | 适配三段式桌面导航；链接使用 Menu 语义 |
| 移动导航 | Drawer + Menu | 适合完整侧栏导航；页面全部内容必须处于 Drawer content 内 |
| 用户/反应菜单 | 点击触发 Dropdown + Menu | 支持键盘/点击；禁用 hover-only 和 CSS focus-only 方案 |
| Feed/评论/通知/关系 | List/ListRow | 连续信息行最符合内容优先；父级唯一外框与全宽分隔线 |
| 治理/审计 | Table 或 List | 字段横向比较用 Table；移动端局部横向滚动；卡片不是替代方案 |
| 一级模块 | 项目 Panel（布局工具） | 使用单层 2px 线框；禁止 Card，防止嵌套盒和阴影 |
| 表单 | Fieldset + Input/Textarea/Select/Validator | 原生字段分组、字段错误和约束状态清晰；禁用 ghost 输入 |
| 页面分区 | Link Tabs | URL 可深链；使用非盒式/非抬升式样式，键盘方向键导航 |
| 分页 | Join + Button | 形成连续页码控制；游标列表仍用普通 Button |
| 确认与治理动作 | 原生 dialog + Modal | 需要焦点限制和 Escape；危险操作不使用 popover |
| 原位状态 | Alert | 重要错误/恢复入口可保留在模块内；小屏响应式纵向 |
| 短成功反馈 | Toast 包裹 Alert | 仅非关键反馈，不承载错误恢复 |
| 加载 | Skeleton / Loading | 模块加载用 Skeleton，按钮提交用 Loading |
| 元信息 | Badge / Avatar / Breadcrumbs | 状态、人物缩略图与层级导航；Badge 不滥用主色 |

daisyUI 只允许在实现的 UI 适配层出现；静态 HTML 原型没有组件编译边界时，可直接使用官方 class，但 DOM 应按未来项目组件边界分区和命名，便于迁移。

## 9. 视觉与响应式规范

- 最大宽度 1200–1280px；桌面主栏自适应、右栏 300–340px、间距 20–24px。
- `768–1023px` 优先单栏；`<768px` 使用移动 Navbar + Drawer；320px 可完成所有核心任务。
- 所有一级模块、导航、抽屉、列表、表格、表单使用明显 2px 语义线框；`--depth: 0`、`--noise: 0`。
- 不使用阴影、渐变、玻璃、纹理、发光、悬浮层级或嵌套四边框。
- 不使用 Card、Chat、Stat、Hero、Carousel、Stack、FAB、Dock、Hover-3D、Aura、盒式/抬升式 Tabs、ghost 输入或 hover-only Dropdown。
- 列表只有父容器外框，行使用父级贯通分隔线；行无 margin/gap/圆角/阴影。
- 回复只缩进内容，不缩进行和分隔线。
- 颜色以 `base-*` 为主体，状态使用 info/success/warning/error；每页最多一个最重要主操作使用 primary。
- 不硬编码灰/红等 Tailwind 颜色；必须使用 daisyUI 语义颜色保证浅色/深色可读。
- 触控目标至少 44×44px；表格/代码只允许局部横向滚动，不得整页横向滚动。

## 10. 关键页面布局验收

### 10.1 P04 帖子详情

- 左栏顺序固定：面包屑 → 标题/元信息 → 正文 → 互动 → 评论编辑 → 评论列表。
- 标题与正文在同一个 Panel 内，以贯通分隔线分区。
- 评论是独立 List Panel；回复缩进内容但分隔线全宽。
- 帖子锁定时评论表单替换为原因说明；隐藏内容不泄露原文。
- 页面任何位置都没有帖子/评论编辑、删除、撤回。

### 10.2 P24 编辑与发布

- 左栏只有一个表单 Panel，不再包 Card；右栏仅节点规则、草稿状态和不可变提示。
- 编辑/预览可键盘切换，预览与正式渲染采用同一安全规则。
- 附件有上传中、完成、拒绝、扫描、移除状态；未引用附件可清理。
- 发布前 O05 明示不可编辑删除；发布中幂等防重；失败不丢草稿。

### 10.3 P25 通知

- 一个列表外框，未读通过文字/图标/字重共同标识，不只靠颜色。
- 单条已读和全部已读可操作；深链准确定位评论。
- 目标隐藏、被屏蔽、无权限或不存在时安全降级，禁止死链。

### 10.4 P39–P42 治理

- 队列的核心是状态、对象、理由、时间、负责人和优先级；移动端可切换为语义 List。
- 案件页内容只读，治理动作全部要求理由并记录审计。
- 不出现编辑/删除内容按钮；危险动作不能仅由图标表达。
- 403、空队列、部分加载、限流、目标陈旧、申诉逆转全部有状态。

## 11. 可访问性与原型交互验收

- 每页一个 H1，标题层级连续；使用 header/nav/main/aside/section/article/form/list/table 的正确语义。
- 所有按钮和链接真实可点击：页面跳转使用有效相对地址；弹窗/Drawer/Dropdown 可开关；筛选/Tabs/分页有可见选中状态。
- 不允许 `href="#"`、无处理器按钮或点击后无反馈；静态暂态动作至少更新本页可见状态并可撤销/继续。
- Drawer、Dropdown、Dialog 管理焦点进入、限制（Dialog）、Escape 和恢复。
- Tabs 支持左右方向键、Home/End 与 `aria-controls`；分页和当前位置使用明确可访问名称。
- 所有输入有可见 label；字段错误与字段关联；动态结果使用适度 live region。
- 浅色、深色、键盘、读屏、reduced-motion、320px、200% 缩放均可完成核心流程。
