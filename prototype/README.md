# X2Post 静态交互原型

本目录是依据《轻量级社区产品方案.md》《轻量级社区_UI设计描述文档.md》以及本轮节点发现、金币经济与社区游戏研究实现的全页面静态原型。

## 运行

在仓库根目录执行：

```bash
python3 -m http.server 4173 -d prototype
```

然后访问 `http://127.0.0.1:4173/`。原型使用 daisyUI 5 和 Tailwind CSS 4 CDN，无需安装依赖或构建。页面采用 hash router，因此也可直接打开 `index.html` 浏览；建议用本地 HTTP 服务获得最稳定的行为。

## 页面范围

全部业务页面均由 `index.html` 的静态应用壳承载，路由模板和演示交互位于 `assets/app.js`，daisyUI 适配层位于 `assets/ui.js`。原有 42 个 P0/P1 页面继续保留，并扩展下列完整闭环：

- 发现：Feed、节点目录/详情、帖子/评论、搜索、标签、公开用户与关系分区、403/404。
- 认证：登录、注册、邮箱验证/结果、找回/重置密码、登录任务恢复。
- 登录用户：关注 Feed、收藏、草稿/编辑器、通知、举报列表/详情、屏蔽列表、本人主页。
- 设置：资料、隐私、通知、安全、修改密码、设备会话。
- 治理：申诉列表/新建/详情、审核队列/案件、申诉处理、审计日志。
- 节点发现：Feed 顶部始终可见、自然换行的一级节点入口；选中父级后直接展开二级节点，覆盖父级聚合、两级筛选、目录/详情、发布与搜索路径、空状态和无效路径恢复。
- 低成本贡献：Feed/节点上下文只提供一个轻发布入口；Markdown 下方可独立选择节点，也可携带当前内容与节点转为带标题的长文。
- 金币：钱包、四类余额、个人账本、固定档位悬赏、经济规则、公共透明度、通知设置、风险案件、Controller 控制台、人工调整和三方对账关账。
- 社区旅程：可选新手引导、共建大厅、任务板/详情、个人成长路径、贡献记录、贡献收藏、共建季/详情、节点公开项目、休息设置和状态验收。
- 状态验收：`#/prototype/states` 汇总 Loading、Empty、Partial、Error、401、403、404、429、Conflict 等状态。

## 演示说明

- 头像菜单中的“切换为访客/登录用户”可测试匿名与登录状态。
- 收藏、反应、关注、评论、草稿、通知、举报、申诉、会话撤销和治理动作均有可见反馈。
- 发布前会弹出不可变确认；已发布帖子和评论没有编辑、删除或撤回入口。
- 节点只读；没有节点管理、私信、会员、支付、现金兑换、推荐/热门或个人排行榜入口。
- 金币不可购买、提现、自由转账，也不赋予曝光或治理权；任务只产生质量候选事件，金币服务仍需独立核验和结算。
- 社区旅程无连续签到、掉级、过期未领损失或倒计时焦虑，用户可以暂停并隐藏游戏层。
- 左栏为核心任务，右栏仅显示真实 P1 上下文；移动端自动切换为单栏 Drawer。

金币与游戏是本轮新增的静态产品探索。进入生产开发前，应单独建立新的 OpenSpec change，完成权限、风控、账务一致性、事件模型和滥用测试的评审。
